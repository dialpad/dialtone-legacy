{% header "h2", "Accessibility" %} {% paragraph %}Avoid using toast for critical information since toast disappears automatically and make sure to provide enough time to read the message and act consequently. For best accessible user experience, the amount of time a toast displays for should be user configurable.{% endparagraph %} {% paragraph %}Using {% code %}role="alert"{% endcode %}, it sets {% code %}aria-live="assertive"{% endcode %} which means it will immediately interrupt anything currently being read by the screen reader, so use it for things that require immediate attention such as:{% endparagraph %} {% ul %}
  • An invalid value was entered into a form field
  • The user's login session is about to expire
  • The connection to the server was lost, local changes will not be saved
    • {% endul %} {% paragraph %}Meanwhile {% code %}role="status"{% endcode %} implies {% code %}aria-live="polite"{% endcode %} which means the toast will be read out after what's currently being has finished.{% endparagraph %} {% for i in toast.accessible %} {% assign item = i.item %} {% assign apply = i.applies %} {% assign desc = i.description %} {% endfor %}
